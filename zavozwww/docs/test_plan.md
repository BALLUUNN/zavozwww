# План UI-тестирования проекта FilmBuddy

## Введение
Данный документ описывает сценарии ручного тестирования пользовательского интерфейса (UI) веб-приложения FilmBuddy. Тесты фокусируются на проверке пользовательских путей (User Journeys) через браузер, имитируя реальные действия пользователя.

**Окружение:**
- Адрес приложения: `http://127.0.0.1:5500` (или актуальный адрес Frontend-сервера)

---

## Сценарий 1: Регистрация и настройка профиля

**Цель:** Проверить путь нового пользователя от регистрации до заполнения анкеты.

**Предусловия:**
- Приложение открыто в браузере.
- Пользователь не авторизован (видна форма входа/регистрации).
- Все микросервисы (Auth, Social, RecSystem, Gateway) запущены и доступны.
- Сервис Mailpit (или аналог) запущен для перехвата email-сообщений.
- Используемый для регистрации Email ранее не был зарегистрирован в системе.
- Используемый Username свободен.

**Шаги:**
1.  **Регистрация:**
    *   Нажать ссылку/кнопку "Регистрация" на главной странице.
    *   Заполнить форму регистрации: ввести уникальный Username, Email и Пароль.
    *   Нажать кнопку "Зарегистрироваться".
    *   В появившемся окне ввода кода ввести код подтверждения (полученный из логов сервера или Mailpit).
    *   Нажать "Подтвердить".
2.  **Вход в систему:**
    *   После успешной верификации ввести Email и Пароль в форму входа.
    *   Нажать кнопку "Войти".
3.  **Редактирование профиля:**
    *   Перейти в раздел "Профиль" через навигационное меню.
    *   Нажать кнопку "Редактировать профиль" (или иконку настроек).
    *   Заполнить поля: Имя, Фамилия, Возраст, Город, О себе.
    *   Нажать кнопку "Сохранить".

**Ожидаемый результат:**
- Регистрация проходит успешно, код принимается.
- Вход выполняется, происходит перенаправление на главную страницу.
- На странице профиля отображаются введенные данные (Имя, Город и т.д.).

## Сценарий 2: Социальное взаимодействие (Поиск и добавление друзей)

**Цель:** Проверить механику отправки и принятия заявок в друзья между двумя пользователями.

**Предусловия:**
- В системе зарегистрированы два пользователя: **User A** и **User B**.
- **User A** и **User B** — это разные учетные записи (разные email/username).
- **User A** и **User B** еще не являются друзьями.
- Между **User A** и **User B** нет активных (необработанных) заявок в друзья.
- **User A** авторизован в браузере.
- Сервис SocialService доступен и корректно обрабатывает запросы.

**Шаги:**
1.  **Отправка заявки (User A):**
    *   Перейти в раздел "Друзья" (или "Поиск людей").
    *   В поле поиска ввести имя **User B**.
    *   В результатах поиска найти карточку **User B**.
    *   Нажать кнопку "Добавить в друзья" на карточке пользователя.
    *   Выйти из аккаунта (нажать кнопку "Выход" в меню).
2.  **Принятие заявки (User B):**
    *   Авторизоваться под **User B**.
    *   Перейти в раздел "Друзья".
    *   Переключиться на вкладку "Входящие заявки" (или раздел уведомлений).
    *   Увидеть заявку от **User A** и нажать кнопку "Принять".
3.  **Проверка списка друзей:**
    *   Переключиться на вкладку "Мои друзья".
    *   Убедиться, что **User A** отображается в списке.

**Ожидаемый результат:**
- Заявка успешно отправляется (кнопка меняет состояние на "Заявка отправлена").
- У **User B** появляется входящая заявка.
- После принятия, пользователи появляются друг у друга в списках друзей.

## Сценарий 3: Поиск фильмов, Оценка и Рекомендации

**Цель:** Проверить основной функционал сервиса: поиск контента, выставление оценок и получение рекомендаций.

**Предусловия:**
- Пользователь авторизован.
- Сервисы RecSystem и SocialService запущены.
- База данных фильмов не пуста (содержит тестовые данные).
- Выбранный для теста фильм еще не был оценен текущим пользователем.

**Шаги:**
1.  **Поиск фильма:**
    *   Перейти в раздел "Поиск фильмов".
    *   Ввести название фильма (например, "Star") или выбрать жанр из выпадающего списка.
    *   Нажать кнопку "Найти".
2.  **Просмотр и оценка:**
    *   Кликнуть по карточке любого найденного фильма для перехода к деталям.
    *   На странице фильма выбрать оценку (звезды от 1 до 5).
    *   Ввести текст отзыва в поле комментария.
    *   Нажать кнопку "Оценить".
3.  **Проверка истории оценок:**
    *   Перейти в раздел "Мои оценки" (или в Профиль -> Оценки).
    *   Убедиться, что оцененный фильм появился в списке с правильной оценкой.
4.  **Получение рекомендаций:**
    *   Перейти в раздел "Рекомендации".
    *   Дождаться загрузки списка фильмов.

**Ожидаемый результат:**
- Поиск возвращает список релевантных фильмов.
- Оценка сохраняется без ошибок, появляется уведомление об успехе.
- Фильм отображается в истории оценок.
- Страница рекомендаций отображает список фильмов, сформированный на основе предпочтений.

## Сценарий 4: Комплексный проход (Happy Path)

**Цель:** Быстрая проверка работоспособности всей системы одним сквозным сценарием.
- Все микросервисы (Auth, Social, RecSystem) работают штатно.
- Сервис Mailpit доступен для получения кодов подтверждения.
- В системе существует хотя бы один другой пользователь для теста социального взаимодействия.
- В базе данных фильмов есть фильм с названием "Inception" (или аналог).

**Предусловия:**
- Серверная часть запущена, база данных чистая (или используются уникальные данные).

**Шаги:**
1.  Зарегистрировать нового пользователя "DemoUser" и войти в систему.
2.  Перейти в поиск, найти фильм "Inception" (или любой существующий).
3.  Поставить фильму оценку 10.
4.  Перейти в раздел "Рекомендации" и проверить, что список загрузился.
5.  Перейти в поиск друзей, найти другого пользователя и отправить заявку.
6.  Выйти из системы.

**Ожидаемый результат:**
- Пользователь успешно проходит весь путь без критических ошибок UI.
- Интерфейс отзывчив, переходы между страницами работают корректно.
