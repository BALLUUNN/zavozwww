## План разработки и тестирования

### 1. Стратегия тестирования
Проект использует пирамиду тестирования с акцентом на Unit и Integration тесты для обеспечения надежности микросервисной архитектуры.

#### 1.1. Модульное тестирование (Unit Testing)
*   **Цель:** Проверка изолированной бизнес-логики и доменных сущностей без внешних зависимостей.
*   **Инструменты:** Стандартная библиотека `testing`, библиотека утверждений `testify/assert`, мокирование `testify/mock`.
*   **Объекты тестирования:**
    *   **AuthService:**
        *   Доменные сущности (`user.go`, `userProfile.go`).
        *   Сервисный слой (`user_service.go`, `user_profile_service.go`) с моками репозиториев.
        *   Вспомогательные пакеты (`emailSender`).
    *   **SocialService:**
        *   Доменная логика (`social_profile.go`).
        *   Бизнес-логика (`social_service.go`) с моками `SocialRepository` и `UserServiceClient`.

#### 1.2. Интеграционное тестирование (Integration Testing)
*   **Цель:** Проверка взаимодействия компонентов с базой данных и HTTP-транспортом.
*   **Инструменты:** `httptest` для API, тестовая БД PostgreSQL.
*   **Объекты тестирования:**
    *   **Repositories:** Проверка корректности SQL-запросов и маппинга данных (`user_repo.go`, `social_profile_repo.go`).
    *   **HTTP Handlers:** Тестирование маршрутизации и валидации запросов в `rout.go` (используя `httptest.NewRecorder`).

#### 1.3. Системное тестирование (System / E2E)
*   **Цель:** Проверка сквозных сценариев и межсервисного взаимодействия в среде Docker.
*   **Сценарии:**
    *   Взаимодействие `SocialService` -> `AuthService` (проверка существования пользователя через `UserClient`).
    *   Маршрутизация запросов через API Gateway (Nginx).
